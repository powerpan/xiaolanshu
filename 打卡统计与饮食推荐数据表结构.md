# 打卡统计、训练计划记录与饮食推荐数据表结构

本文档根据当前项目代码整理，覆盖训练打卡统计、训练计划动作完成记录和饮食推荐功能涉及的数据表、字段含义和接口返回结构。

## 一、训练打卡统计功能

训练打卡统计功能使用独立数据表 `fitnesscheckins` 保存用户每日训练记录。后端根据该表计算累计打卡天数、连续打卡天数、本月打卡天数、累计训练时长、平均训练时长和最近打卡记录。

### 1. fitnesscheckins 表结构

| 字段名 | 数据类型 | 约束 | 字段说明 |
|---|---|---|---|
| id | INT / SERIAL | 主键，自增 | 打卡记录编号 |
| username | VARCHAR(50) | NOT NULL | 用户名，用于关联用户 |
| checkin_date | DATE | NOT NULL | 打卡日期 |
| duration_minutes | INT | DEFAULT 0 | 本次训练时长，单位为分钟 |
| mood | VARCHAR(50) | 可空 | 训练状态，例如“状态不错”“轻松”“有挑战”“需要恢复” |
| note | TEXT | 可空 | 用户填写的训练备注 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 记录创建或更新时间 |

### 2. 唯一索引

| 索引名 | 字段 | 说明 |
|---|---|---|
| uk_fitnesscheckins_username_date | username, checkin_date | 限制同一用户同一天只能保存一条打卡记录 |

### 3. 建表语句

H2 版本：

```sql
CREATE TABLE IF NOT EXISTS fitnesscheckins (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    checkin_date DATE NOT NULL,
    duration_minutes INT DEFAULT 0,
    mood VARCHAR(50),
    note TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX IF NOT EXISTS uk_fitnesscheckins_username_date
    ON fitnesscheckins(username, checkin_date);
```

openGauss 版本：

```sql
CREATE TABLE fitnesscheckins (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    checkin_date DATE NOT NULL,
    duration_minutes INT DEFAULT 0,
    mood VARCHAR(50),
    note TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX uk_fitnesscheckins_username_date
    ON fitnesscheckins(username, checkin_date);
```

### 4. 统计字段来源

| 统计字段 | 类型 | 计算来源 |
|---|---|---|
| totalDays | Integer | 当前用户在 `fitnesscheckins` 中的总记录数 |
| currentStreak | Integer | 根据当前用户所有打卡日期倒序计算当前连续打卡天数 |
| bestStreak | Integer | 根据当前用户所有打卡日期计算历史最佳连续打卡天数 |
| monthDays | Integer | 当前自然月内的打卡记录数 |
| totalMinutes | Integer | 当前用户所有 `duration_minutes` 的总和 |
| averageMinutes | Integer | `totalMinutes / totalDays` |
| recentCheckins | ArrayList<FitnessCheckin> | 当前用户最近 42 条打卡记录 |

## 二、训练计划动作完成记录

训练计划动作勾选结果使用 `plantaskrecords` 保存。它和 `fitnesscheckins` 不同：`fitnesscheckins` 表示用户当天是否打卡，`plantaskrecords` 表示某一天某个训练计划动作是否完成、做了多少组/次以及主观难度。

### 1. plantaskrecords 表结构

| 字段名 | 数据类型 | 约束 | 字段说明 |
|---|---|---|---|
| id | INT / SERIAL | 主键，自增 | 训练计划动作记录编号 |
| username | VARCHAR(50) | NOT NULL | 用户名 |
| plan_date | DATE | NOT NULL | 计划执行日期 |
| daytime | INT | NOT NULL | 第几天训练计划 |
| action_index | INT | NOT NULL | 当天动作序号 |
| actionpattern | VARCHAR(100) | 可空 | 动作模式 |
| actionname | VARCHAR(100) | 可空 | 动作名称 |
| equipment | VARCHAR(50) | 可空 | 器材 |
| completed | BOOLEAN | DEFAULT FALSE | 是否完成 |
| actual_sets | INT | 可空 | 实际完成组数 |
| actual_reps | INT | 可空 | 实际完成次数 |
| difficulty_score | INT | 可空 | 主观难度评分 |
| note | TEXT | 可空 | 动作备注 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### 2. 唯一索引

| 索引名 | 字段 | 说明 |
|---|---|---|
| uk_plantaskrecords_key | username, plan_date, daytime, action_index | 限制同一用户同一天同一个计划动作只有一条记录 |

### 3. 建表语句

H2 版本：

```sql
CREATE TABLE IF NOT EXISTS plantaskrecords (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    plan_date DATE NOT NULL,
    daytime INT NOT NULL,
    action_index INT NOT NULL,
    actionpattern VARCHAR(100),
    actionname VARCHAR(100),
    equipment VARCHAR(50),
    completed BOOLEAN DEFAULT FALSE,
    actual_sets INT,
    actual_reps INT,
    difficulty_score INT,
    note TEXT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX IF NOT EXISTS uk_plantaskrecords_key
    ON plantaskrecords(username, plan_date, daytime, action_index);
```

openGauss 版本：

```sql
CREATE TABLE plantaskrecords (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    plan_date DATE NOT NULL,
    daytime INT NOT NULL,
    action_index INT NOT NULL,
    actionpattern VARCHAR(100),
    actionname VARCHAR(100),
    equipment VARCHAR(50),
    completed BOOLEAN DEFAULT FALSE,
    actual_sets INT,
    actual_reps INT,
    difficulty_score INT,
    note TEXT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX uk_plantaskrecords_key
    ON plantaskrecords(username, plan_date, daytime, action_index);
```

### 4. 训练计划洞察字段来源

| 字段 | 类型 | 计算来源 |
|---|---|---|
| weeklyTarget | Integer | `userprofiles.weeklyfrequency` |
| recordedTasks | Integer | 当前自然周 `plantaskrecords` 记录数 |
| completedTasks | Integer | 当前自然周 `completed = TRUE` 的动作记录数 |
| completionRate | Integer | `completedTasks / recordedTasks` |
| currentStreak | Integer | `fitnesscheckins` 中计算出的当前连续打卡天数 |
| recommendation | String | 根据完成率和记录情况生成的训练建议 |
| recoveryHint | String | 根据打卡状态、连续打卡和完成率生成的恢复提示 |
| nextAdjustment | String | 根据完成率和周频率生成的下一步调整建议 |

## 三、饮食推荐功能

饮食推荐会实时读取用户基础信息、训练档案和饮食偏好生成结果，同时把最近生成的推荐摘要保存到 `nutritionrecommendationhistories`。

主要数据来源为：

- `users.weight`：用户体重，是热量、蛋白质、脂肪和饮水量计算的核心依据。
- `userprofiles.fitnessgoal`：用户训练目标，会影响热量和蛋白质系数。
- `userprofiles.weeklyfrequency`：用户每周训练频率，用于推荐摘要和训练日安排。
- `nutritionpreferences`：用户饮食类型、忌口、预算、外食频率、每日餐次和口味偏好。

### 1. users 表中相关字段

| 字段名 | 数据类型 | 约束 | 与饮食推荐的关系 |
|---|---|---|---|
| id | INT / SERIAL | 主键，自增 | 用户编号 |
| username | VARCHAR(50) | NOT NULL UNIQUE | 用户名，用于查询当前用户 |
| nickname | VARCHAR(50) | 可空 | 用户昵称 |
| password | VARCHAR(100) | NOT NULL | 登录密码哈希 |
| userpic | VARCHAR(1000) | 可空 | 用户头像 |
| createtime | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 用户创建时间 |
| registered | BOOLEAN | DEFAULT FALSE | 用户审核状态 |
| identity | VARCHAR(20) | DEFAULT 'user' | 用户身份 |
| specialty | VARCHAR(200) | 可空 | 用户训练方向或偏好 |
| height | DOUBLE / DOUBLE PRECISION | 可空 | 当前饮食算法未直接使用 |
| weight | DOUBLE / DOUBLE PRECISION | 可空 | 饮食推荐核心计算依据 |

### 2. userprofiles 表中相关字段

| 字段名 | 数据类型 | 约束 | 与饮食推荐的关系 |
|---|---|---|---|
| id | INT / SERIAL | 主键，自增 | 训练档案编号 |
| username | VARCHAR(50) | NOT NULL UNIQUE | 用户名，用于关联用户 |
| fitnessgoal | VARCHAR(50) | NOT NULL | 饮食推荐核心依据，例如“保持健康”“增肌”“减脂”“塑形”“提升力量” |
| weeklyfrequency | INT | NOT NULL | 每周训练频率，会写入推荐结果并影响训练日提醒 |
| equipment | VARCHAR(50) | NOT NULL | 可用器材，当前饮食算法未直接使用 |
| exlevel | VARCHAR(50) | NOT NULL | 训练水平，当前饮食算法未直接使用 |

### 3. nutritionpreferences 表结构

| 字段名 | 数据类型 | 约束 | 与饮食推荐的关系 |
|---|---|---|---|
| id | INT / SERIAL | 主键，自增 | 偏好编号 |
| username | VARCHAR(50) | NOT NULL UNIQUE | 用户名，每个用户一条偏好 |
| diettype | VARCHAR(50) | DEFAULT '均衡饮食' | 饮食类型，例如均衡饮食、高蛋白、低脂、素食友好、控糖 |
| allergies | VARCHAR(300) | 可空 | 忌口或过敏源 |
| budgetlevel | VARCHAR(50) | DEFAULT '中等' | 饮食预算 |
| eatingoutfrequency | VARCHAR(50) | DEFAULT '偶尔外食' | 外食频率 |
| mealcount | INT | DEFAULT 4 | 每日餐次 |
| tastepreference | VARCHAR(100) | DEFAULT '清淡' | 口味偏好 |

### 4. nutritionrecommendationhistories 表结构

| 字段名 | 数据类型 | 约束 | 字段说明 |
|---|---|---|---|
| id | INT / SERIAL | 主键，自增 | 推荐历史编号 |
| username | VARCHAR(50) | NOT NULL | 用户名 |
| targetcalories | INT | 可空 | 生成时的目标热量 |
| proteingrams | INT | 可空 | 生成时的蛋白质目标 |
| carbohydrategrams | INT | 可空 | 生成时的碳水目标 |
| fatgrams | INT | 可空 | 生成时的脂肪目标 |
| preferencesummary | TEXT | 可空 | 生成时的偏好摘要 |
| summary | TEXT | 可空 | 生成时的推荐摘要 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 生成时间 |

### 5. 饮食推荐计算逻辑

| 推荐项 | 计算方式 |
|---|---|
| 基础热量 | `weight * 30` |
| 增肌 / 提升力量 | 基础热量增加 300 kcal，蛋白质系数为 `1.8g/kg` |
| 减脂 / 塑形 | 基础热量减少 250 kcal，蛋白质系数为 `1.7g/kg` |
| 保持健康 | 使用基础热量，蛋白质系数为 `1.5g/kg` |
| 最低热量保护 | 目标热量最低不低于 1300 kcal |
| 蛋白质 | `weight * proteinRatio` |
| 脂肪 | `weight * 0.8` |
| 碳水 | `(targetCalories - proteinGrams * 4 - fatGrams * 9) / 4`，最低 100g |
| 饮水 | `weight * 35ml` |

如果用户体重为空，代码默认按 `65kg` 计算；如果用户训练目标为空，默认按“保持健康”计算。

### 6. 饮食推荐返回结构

饮食推荐结果由接口实时返回 `NutritionRecommendation`，并同步写入一条推荐历史摘要。

| 字段名 | 类型 | 字段说明 |
|---|---|---|
| goal | String | 当前训练目标 |
| weightKg | Double | 当前计算使用的体重 |
| weeklyFrequency | Integer | 当前每周训练频率 |
| targetCalories | Integer | 每日目标热量，单位 kcal |
| proteinGrams | Integer | 每日蛋白质建议摄入量，单位 g |
| carbohydrateGrams | Integer | 每日碳水建议摄入量，单位 g |
| fatGrams | Integer | 每日脂肪建议摄入量，单位 g |
| waterMl | Integer | 每日饮水建议，单位 ml |
| summary | String | 总体饮食建议说明 |
| preferenceSummary | String | 当前偏好摘要 |
| trainingDayTip | String | 训练日饮食提示 |
| restDayTip | String | 休息日饮食提示 |
| meals | ArrayList<String> | 一日饮食结构建议 |
| trainingDayMeals | ArrayList<String> | 训练日餐食模板 |
| restDayMeals | ArrayList<String> | 休息日餐食模板 |
| tips | ArrayList<String> | 饮食执行建议 |
| mealTiming | ArrayList<String> | 训练前后和一天内进餐时机 |
| foodChoices | ArrayList<String> | 食材选择建议 |
| replacements | ArrayList<String> | 食材替换建议 |
| shoppingList | ArrayList<String> | 采购清单 |
| watchouts | ArrayList<String> | 注意事项 |

### 7. 饮食推荐历史返回结构

接口 `/nutrition/history` 返回当前用户最近 10 条 `NutritionRecommendationHistory`。

| 字段名 | 类型 | 字段说明 |
|---|---|---|
| id | Integer | 推荐历史编号 |
| username | String | 用户名 |
| targetCalories | Integer | 目标热量 |
| proteinGrams | Integer | 蛋白质目标 |
| carbohydrateGrams | Integer | 碳水目标 |
| fatGrams | Integer | 脂肪目标 |
| preferenceSummary | String | 偏好摘要 |
| summary | String | 推荐摘要 |
| createdAt | LocalDateTime | 生成时间 |

## 四、功能与数据表对应关系

| 功能模块 | 是否有独立数据表 | 相关表或结构 |
|---|---|---|
| 训练打卡 | 是 | `fitnesscheckins` |
| 打卡统计 | 否，基于打卡表计算 | `fitnesscheckins`、`CheckinStats` |
| 训练计划动作完成记录 | 是 | `plantaskrecords`、`TrainingPlanInsight` |
| 饮食偏好 | 是 | `nutritionpreferences` |
| 饮食推荐 | 部分落库，结果实时计算 | `users`、`userprofiles`、`nutritionpreferences`、`NutritionRecommendation` |
| 饮食推荐历史 | 是 | `nutritionrecommendationhistories`、`NutritionRecommendationHistory` |
