# 小蓝鼠功能验收清单

本文档用于每次大改后快速确认项目功能没有回退。验收前请先启动后端和前端。

## 启动与自动验证

后端测试：

```sh
cd fitnessGuidance
MAVEN_USER_HOME=.m2 ./mvnw test
```

前端构建：

```sh
cd vue-project
npm run build
```

Windows 同事首次部署和排错流程见：

```text
docs/Windows部署操作流程.md
```

饮食推荐、文章生成发布、训练打卡、训练复盘和管理端的实现说明及详细测试用例见：

```text
docs/功能实现与测试用例.md
```

本地访问：

```text
前端：http://127.0.0.1:5173
后端：http://127.0.0.1:8080
```

## 普通用户端

测试账号：

```text
demo / demo123
```

验收项：

- 登录成功后进入普通用户端，侧边栏只显示首页、训练打卡、训练复盘、饮食建议、健身需求、训练计划、动作指导、公告中心、文章广场；个人资料从顶部身份标识进入。
- 侧边栏在桌面端固定不随主内容滚动，菜单区域不出现左右或上下滚动条；移动端菜单自动换行。
- 左上角品牌展示小蓝鼠纯色标识和名称，页面保持原有暖色调。
- 左下角音乐光碟可拖动，点击可播放/暂停“橙色制高点”，播放时有旋转和光效，歌名只在鼠标悬停时显示。
- 首页核心卡片、热力图、动作提示、文章和公告区域在常见 PC 浏览器宽度下不越界、不重叠。
- 个人资料可以保存昵称、擅长方向、身高、体重；密码留空时不强制修改密码。
- 健身需求可以保存目标、每周频率、器材、训练水平；保存后训练计划和饮食建议能刷新。
- 训练计划可以按第几天切换，动作列表包含动作名称、动作模式、器材、步骤、要点、组次和休息时间。
- 训练计划动作卡片展示推荐理由、今日强度、进阶建议、主肌群、常见错误、禁忌和替代动作。
- 训练计划展示训练周期、当前阶段、恢复日、本周完成率、连续打卡、恢复提示和下一步调整建议。
- 训练计划动作卡片中可选择同动作模式的替换动作，替换后步骤、要点、肌群和风险提示随动作更新。
- 训练计划后续天数可以提前查看；未完成前序计划日时，后续计划只能查看，不能勾选动作或提前打卡。
- 训练计划动作完成状态会保存到后端，刷新页面后当天动作勾选状态仍保留。
- 训练计划内勾选动作后，可以按完成情况生成今日打卡。
- 动作指导可以按动作模式和器材搜索；动作百科可以筛选动作，不展示管理员编辑入口。
- 动作指导详情展示难度、主肌群、辅助肌群、常见错误、禁忌提醒、适合人群和替代动作。
- 当前内置动作的详情字段按具体动作区分，例如俯卧撑、杠铃卧推和坐姿推胸机不共用同一套风险提示和替代动作。
- 动作指导图片只在动作指导页展示；没有图片的动作显示占位，不使用错误的自动绘制图片。
- 训练打卡可以保存时长、状态、备注；保存后累计天数、连续打卡和历史记录刷新。
- 训练复盘展示准备度、最佳连续、平均时长、最近 35 天热力和教练建议。
- 饮食建议展示目标热量、蛋白质、碳水、脂肪、饮水，并把具体内容浓缩为推荐餐单、食材与替换、场景与注意三块。
- 饮食建议页不展示偏好设置表单，不出现“保存偏好”等用户侧维护入口。
- 公告中心只展示已发布公告，点击可进入详情页。
- 文章广场支持标题和主题搜索，点击长文可进入详情页并返回列表。

## 管理端

测试账号：

```text
admin / admin123
```

验收项：

- 登录成功后进入管理端，侧边栏只显示内容管理、动作库管理、饮食模板、用户管理。
- 管理端侧边栏同样不出现左右或上下滚动条，低高度窗口下仍能完整看到管理菜单。
- 管理端不显示普通用户训练计划、饮食建议、打卡等使用型入口。
- 内容管理中，文章列表显示在上方，新增文章表单显示在下方。
- 文章可以新增、进入详情、弹窗编辑和删除；编辑内容不会回写到新增表单。
- 公告列表显示在上方，新增公告表单显示在下方。
- 公告可以新增、进入详情、弹窗编辑、删除、保存草稿、载入草稿、清空草稿。
- 动作库管理中，动作列表显示在上方，新增动作表单显示在下方。
- 动作库支持按动作模式、器材筛选，并支持“只看未自定义图片”“只看不完整”“缺步骤”“缺要点”“缺常见错误”“缺替代动作”筛选。
- 动作可以新增、弹窗编辑、删除。
- 动作新增和编辑支持维护难度、主肌群、辅助肌群、常见错误、禁忌提醒、适合人群和替代动作。
- 动作图片由管理员上传；上传后图片路径保存到动作记录，文件保存到后端上传目录。
- 饮食模板管理可以维护餐单模板、食材、替换规则和外食场景。
- 新增或编辑饮食模板后，普通用户饮食建议中的餐单模板、替换规则和场景建议会更新。
- 用户管理可以添加用户、审核注册、编辑用户资料、删除用户。
- 管理员编辑用户时使用弹窗，不把编辑内容回写到新增用户表单。

## 数据库与上传目录

- 默认本地开发使用 H2 文件库，数据库文件生成在 `fitnessGuidance/data`。
- openGauss 初始化脚本位于 `fitnessGuidance/database/opengauss-init.sql`。
- 训练打卡记录的计划天数保存于 `fitnesscheckins.plan_day` 字段。
- 训练计划动作完成记录保存于 `plantaskrecords` 表。
- 训练周期保存于 `trainingcycles` 表。
- 计划调整记录保存于 `planadjustmentrecords` 表。
- 动作替换关系保存于 `exercisealternatives` 表。
- `nutritionpreferences` 为后端兼容保留表，当前普通用户端不展示单独的偏好设置表单。
- 食材、餐单模板、替换规则和外食场景分别保存于 `fooditems`、`mealtemplates`、`foodreplacements`、`eatingscenarios` 表。
- 管理员上传的动作图片保存到 `fitnessGuidance/uploads/exercise-guides`。
- 仓库只保留上传目录占位文件 `.gitkeep`，不提交运行期上传图片。

## 回归重点

- 普通用户不能访问管理端入口。
- 管理员端只显示管理相关功能。
- 后端接口路径保持不变，前端结构调整后不能改动已有接口名称。
- 页面在 1280px、1440px、MacBook Air 常见浏览器宽度下不出现明显越界、重叠、白底白字。
